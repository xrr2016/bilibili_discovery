# Bilibili Discovery 数据结构与数据库设计文档

## 一、设计目标

本系统是一个用于 **视频兴趣分析与知识管理的浏览器扩展系统**。

核心能力：

1. 用户视频行为记录
2. 兴趣分析
3. 标签语义管理
4. 视频笔记与知识管理
5. 收藏与搜索
6. LLM 对话搜索
7. 推荐系统
8. 多平台支持（B站 / YouTube）

因此数据库设计必须满足：

* 高扩展性
* 可分析性
* 支持 AI 语义计算
* 支持多平台
* 支持行为日志

数据库使用：

```
IndexedDB
```

设计原则：

1. **行为数据独立**
2. **分析数据可重建**
3. **语义数据独立**
4. **平台数据隔离**
5. **避免重复数据**

# 四、Content Layer（内容层）

该层记录视频与创作者基础信息。

这些数据为 **行为分析和笔记系统提供基础信息**。

---

# 4.1 Creator（UP主 / Channel）

描述视频创作者。

数据来源：

* B站关注列表
* YouTube订阅列表

数据结构：

```
Creator

creator_id 平台中的唯一id
platform    平台
name        名字
avatar      头像图片数据
is_logout      是否已注销?
description     简介
created_at      创建时间
follow_time     关注时间
follow_time     是否关注/订阅
tag权重列表      记录当前up的tag以及每个标签的权重
```

用途：

* 视频归属
* 兴趣分析（UP主偏好）
* UP排名
* 推荐系统

关联关系：

```
Creator
  └── Video
        └── WatchEvent
```


```

---
tag 权重列表

包含tag id 

created_at

source    // user或者system,user手动添加的是可编辑的标签,system添加的是带有计数器的标签

count: number; // 计数器，用于统计system添加的标签优先级

这个信息用于给up打上标签和权重,手动添加的标签权重是0,系统添加的则在后续可以不断的增长
```

```
# 五、Behavior Layer（行为数据）

行为数据是 **兴趣分析系统的核心基础数据**。

该层记录所有用户行为。

---

# 5.1 WatchEvent（观看事件）

记录用户每一次观看行为。

数据结构：

```
WatchEvent

event_id

platform
video_id
creator_id

watch_time

watch_duration

video_duration

progress

source
```

字段说明：

| 字段             | 说明         |
| -------------- | ---------- |
| watch_time     | 观看时间       |
| watch_duration | 实际观看时长     |
| video_duration | 视频总时长      |
| progress       | 观看比例       |
| source         | 来源（推荐/搜索等） |

用途：

* 兴趣分析
* 推荐算法
* UP排名
* 用户观看时间统计

关系：

```
WatchEvent
 ├── Video
 └── Creator
```

大分区（标签聚合）

export interface Category {
  id: string; // 分区ID
  name: string; // 分区名称

---

# 6.2 TagAlias（标签映射）

用于标签合并。

解决：

```
AI
人工智能
machine learning
```

数据结构：

```
TagAlias

target_tag

```

用途：

* Tag统一
* Tag合并
* AI计算
输入的tag与TagAlias中的内容进行匹配,匹配成功则输出target_tag

语义上几乎相同的标签经过这里可以转换为固定稳定的标签
核心原理是维护一个映射表,这个表如果命中即可完成转换,
如果没有命中,尝试使用嵌入式矩阵,并将结果更新写回映射
如果矩阵不可用,则尝试使用近似匹配,
如果匹配失败,则输出原输入.
```
