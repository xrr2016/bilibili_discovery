# 兴趣分析系统实现规划说明

## 一、可行性分析

**结论：完全可以实现**

基于当前数据库架构，系统已经具备了实现兴趣分析系统所需的所有基础能力：
1. ✅ 完整的数据类型定义（InterestScore、InterestNode、InterestHistory、UserInterestProfile）
2. ✅ 完善的接口定义（IInterestScoreRepository、IInterestNodeRepository、IInterestHistoryRepository等）
3. ✅ 基础实现层（InterestScoreRepository已实现）
4. ✅ 行为数据采集能力（WatchEvent、InteractionEvent、SearchEvent）
5. ✅ 语义数据支持（Tag、Category、TagEmbedding）

## 二、实现方案

### 5.1 用户兴趣权重模型

**实现路径：**
1. **数据来源**
   - 从WatchEvent获取观看行为（时长、进度、完整性）
   - 从InteractionEvent获取互动行为（点赞、评论、分享等）
   - 从Video获取视频标签信息
   - 从Tag获取标签元数据

2. **计算逻辑**
   - 短期兴趣：基于最近N天（如7天）的行为数据
   - 长期兴趣：基于历史所有行为数据，采用时间衰减算法
   - 综合兴趣：短期和长期兴趣的加权组合

3. **实现步骤**
   - 在implementations中创建`interest-calculator.impl.ts`
   - 实现兴趣计算核心算法（时间衰减、行为权重、标签聚合）
   - 利用已有的InterestScoreRepository存储结果
   - 创建定时更新机制（如每日更新）

4. **查询能力**
   - 复用InterestScoreRepository的getInterestScore、getInterestScores、getTopInterests等方法
   - 在implementations中添加聚合方法，支持按时间范围、分数类型等维度查询

### 5.2 兴趣星球系统

**实现路径：**
1. **数据结构**
   - 利用已有的InterestNode类型
   - 需要实现InterestNodeRepository（接口已定义，实现未完成）

2. **构建逻辑**
   - 基于标签相似度（TagEmbedding）构建节点关联
   - 基于Category的层级结构初始化节点树
   - 基于用户兴趣权重计算节点权重

3. **实现步骤**
   - 在implementations中创建`interest-node-repository.impl.ts`
   - 实现节点创建、更新、查询、删除等基础操作
   - 创建`interest-graph-builder.impl.ts`实现兴趣图构建逻辑
   - 实现节点权重动态更新机制

4. **可视化支持**
   - 利用InterestNode的color和icon字段
   - 通过getNodeTree方法获取完整层级结构
   - 实现节点路径查询（getNodePath）和子节点查询（getChildNodes）

### 5.3 兴趣时间变化分析

**实现路径：**
1. **数据记录**
   - 利用已有的InterestHistory类型
   - 需要实现InterestHistoryRepository（接口已定义，实现未完成）

2. **时间衰减**
   - 实现指数衰减函数：score = initialScore * e^(-λt)
   - 在兴趣计算时应用衰减因子
   - 支持配置衰减速率参数

3. **趋势分析**
   - 利用InterestHistoryRepository的getInterestTrend方法
   - 计算移动平均、变化率等指标
   - 实现趋势分类（上升、平稳、下降）

4. **实现步骤**
   - 在implementations中创建`interest-history-repository.impl.ts`
   - 实现历史记录的增删查改
   - 创建`interest-trend-analyzer.impl.ts`实现趋势分析
   - 定期记录兴趣快照（如每日一次）

## 三、技术要点

### 1. 数据分层
- **持久化层**：IndexedDB存储InterestScore、InterestNode、InterestHistory
- **计算层**：implementations中的兴趣计算、图构建、趋势分析
- **展示层**：UI组件调用实现层方法获取数据

### 2. 性能优化
- 批量操作：使用updateInterestScores、recordInterestHistories等批量方法
- 索引优化：确保INTEREST_SCORES的score索引、INTEREST_NODES的parentId索引
- 缓存策略：对频繁访问的兴趣数据实现内存缓存

### 3. 数据一致性
- 兴趣分数更新时同步更新InterestHistory
- 节点权重变化时更新相关InterestScore
- 使用事务确保批量操作的原子性

### 4. 扩展性
- 支持自定义兴趣计算参数（时间窗口、衰减速率、权重配置）
- 支持动态调整兴趣星球结构
- 支持多种趋势分析算法

## 五、注意事项

1. **遵循代码规范**
   - 所有数据操作通过implementations层完成
   - 不在UI层直接操作数据库
   - 不随意扩展长期数据模型

2. **模块职责清晰**
   - 兴趣计算：interest-calculator.impl.ts
   - 图构建：interest-graph-builder.impl.ts
   - 趋势分析：interest-trend-analyzer.impl.ts
   - 数据访问：各Repository实现

3. **数据边界明确**
   - InterestScore：存储兴趣权重
   - InterestNode：存储兴趣结构
   - InterestHistory：存储历史变化
   - UserInterestProfile：存储综合画像

4. **性能考虑**
   - 历史数据定期清理（如保留1年）
   - 批量操作优化
   - 合理使用索引
   - 避免频繁的全量计算
