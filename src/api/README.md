# `src/api`

本目录负责封装与 Bilibili 相关的外部接口访问逻辑。

主要职责：
- 统一管理请求入口、参数拼装、限流与鉴权头
- 提供面向业务的 API 方法，例如获取关注列表、UP 信息、视频列表、标签等
- 隔离外部接口细节，避免业务层直接拼接 URL

使用约定：
- 这里负责“怎么请求外部接口”
- 不负责业务决策、分类策略、数据库持久化
- 业务层应通过这里拿到原始或轻度整理后的平台数据，再交给 `engine` 或 `background`

主要文件：
- `bili-api.ts`：B 站接口主封装
- `bili-config.ts`：本地 API 配置
- `bili-config.example.ts`：配置示例
